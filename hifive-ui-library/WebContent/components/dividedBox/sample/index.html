ボックス区切り移動サンプル(ボックス追加機能、リセット機能あり)

使用方法・制限事項
  • ここではdividedBoxControllerをサブコントローラーとして使用しています。
    class=sampleの要素がルート要素、配下のclass=_dividedBoxの要素がdividedBoxControllerの対象です。
  • _dividedBoxの要素についてはこちらと同様です。
  • insertボタンは、dividedBoxControllerのinsertメソッドを実行することにより、
    2番目の要素の後ろに要素を追加して、その間に区切り線を生成します。
  • addDiv→ボタンは、_dividedBoxの要素の末尾にDIV要素を追加するのみです。
    その後、refreshボタンを押すと、dividedBoxControllerのrefreshメソッドを実行することにより、
    区切り線のない要素間に区切り線を生成します。

ルート要素は縦幅サイズ200px、両端のボックスはfill指定あり、中央はfill指定なしで縦幅サイズ100pxを指定
ルート要素にfreezeSize指定がなく、横幅の指定がない(autoで100%)ため、ウィンドウサイズ変更時にルート要素と内容がずれる。
refreshボタンを押すことで修正される。

(上との違いは、ルート要素にfreezeSize指定が追加されている点のみ)
ルート要素はfreezeSize指定ありで縦幅サイズ指定なし、中央のボックスに縦幅サイズ100pxを指定
このため、処理前のルート要素の高さは中央のボックスの高さに依存する。
freezeSize指定によりその状態でサイズが固定される。
両端のボックスはfill指定ありなので縦幅が100%となる。

ルート要素はfreezeSize指定なしで横幅サイズ指定なし、両端のボックスはfill指定あり、中央はfill指定なしで横幅サイズ300pxを指定
（このケースは縦幅が動的に変わらないのでウィンドウサイズ変更等で内容のずれは生じない。）
